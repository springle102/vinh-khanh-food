01 • Tổng quan

Bài toán kinh doanh và mục tiêu

Hệ thống được xây dựng để số hóa trải nghiệm tham quan ẩm thực và chuẩn hóa nội dung vận hành cho Phố Ẩm thực Vĩnh Khánh.

01

Bối cảnh

Khu phố ẩm thực có nhiều quán, món đặc trưng và nhu cầu tiếp cận khách trong nước lẫn quốc tế, nhưng dữ liệu giới thiệu hiện dễ phân tán và thiếu chuẩn hóa.

Vấn đề cần giải quyết

  • Thông tin quán, món, ưu đãi và lộ trình chưa có nguồn dữ liệu thống nhất.
  • Thiếu bản dịch và audio guide nhất quán cho khách quốc tế.
  • Quản lý QR, review và media còn rời rạc, khó kiểm soát.

Mục tiêu dự án

  • Xây dựng admin dashboard tập trung.
  • Quản lý nội dung đa ngôn ngữ và audio.
  • Chuẩn hóa dữ liệu backend làm nguồn chuẩn duy nhất.

Kiến trúc tổng quan

Cấu trúc hệ thống dựa trên đồ án hiện tại

Phần này mô tả trung thực kiến trúc source đang có: một `admin-web` React/Vite ở phía frontend và một `backend-api` ASP.NET Core ở phía server.

ARC
Kiến trúc hệ thống
Dòng dữ liệu admin

Đồ án hiện tại không tách thành microservice. Backend là một API duy nhất, nhưng đã được chia thành 6 nhóm nghiệp vụ REST rõ ràng, dùng chung AdminDataRepository và StorageService. Dữ liệu vận hành hiện đang persist trực tiếp về SQL Server và file upload nằm dưới /wwwroot/storage. Hai card cuối là hướng mở rộng hợp lý cho giai đoạn sau.

CONT

content

Places, translations, food items, media assets, SEO, publish status.

MED

media_audio

Audio guides, upload MP3/image, StorageService, static file URL dưới /storage.

AUTH

admin_security

Đăng nhập, RBAC, admin user CRUD, session, managedPlaceId và kiểm soát quyền.

BOOT

bootstrap_dashboard

GET /bootstrap, dashboard summary, hydrate state ban đầu và refresh toàn cục.

QR

qr_routes

QR image/state, route CRUD, featured path và điểm vào cho trải nghiệm quét mã.

OPS

operations

Promotions, review moderation, settings, activity log và tham số vận hành.

DB

data_platform (reserved)

EF Core, SQL Server, migration, backup và tách repository dữ liệu khi mở rộng.

APP

public_experience (reserved)

Public landing/app cho du khách, scan flow hoàn chỉnh, favorites và premium unlock.

Luồng kiến trúc hiện tại admin-web (React + Vite) → REST Controllers → AdminDataRepository + StorageService → SQL Server + /wwwroot/storage

09 • Nhược điểm

Nhược điểm

09

Rủi ro dữ liệu

Backend hiện vẫn persist qua seed SQL thay vì cơ sở dữ liệu quan hệ hoàn chỉnh, nên khả năng mở rộng và kiểm soát đồng thời còn hạn chế.

Rủi ro nội dung

Bản dịch, audio và mô tả điểm đến cần được vận hành nhất quán; nếu quy trình kiểm duyệt lỏng, trải nghiệm du khách có thể không đồng đều.

Rủi ro phân quyền

Nếu role và phạm vi managedPlaceId không được kiểm soát chặt ở backend, người dùng có thể thao tác vượt quyền.

Phụ thuộc hạ tầng

Admin web phụ thuộc vào backend API, storage và dữ liệu seed ban đầu để có thể trình diễn đầy đủ trong môi trường đồ án.

Hướng phát triển hợp lý sau đồ án là chuyển lớp persist hiện tại sang EF Core + SQL Server nhưng vẫn giữ nguyên contract bootstrap/auth/CRUD để frontend không phải thay đổi lớn.
